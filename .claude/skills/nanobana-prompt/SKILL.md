# 나노바나나 프롬프트 생성기

크루즈 상품 데이터를 읽어 Google 공식 가이드 기반으로 최적화된 나노바나나 이미지 프롬프트를 생성한다.

---

## Step 1: 입력 분석

### 상품명 식별

사용자 메시지에서 크루즈 상품을 식별한다. 한글, 영문, 약칭 모두 인식:

| 한글명 | slug | 약칭 |
|--------|------|------|
| 알래스카 | alaska-emerald-princess | 알래스카, alaska |
| 아라비아 | arabia-toscana | 아라비아, 두바이, arabia |
| 북유럽 | north-europe-diadema | 북유럽, 노르웨이, north europe |
| 오키나와/대만 | okinawa-taiwan-bellissima | 오키나와, 대만, okinawa, taiwan |
| 남지중해 / 남부 지중해 | southern-med-fortuna | 남지중해, 산토리니, 에게해 |
| 서지중해 / 서부 지중해 | western-med-seaview | 서지중해, 바르셀로나, 로마 |

상품명이 불명확하면 사용자에게 질문한다. 복수 상품 매칭 시 목록을 보여주고 선택을 요청한다.

### 입력 유형 판별

| 유형 | 판별 기준 | 처리 |
|------|-----------|------|
| **기존 프롬프트** | 영문 50단어 이상, 또는 camera/lens/shot/lighting/film 등 촬영 키워드 포함 | → Step 3-B (최적화 모드) |
| **자연어 설명** | 한국어 짧은 설명, 장면 콘셉트 등 | → Step 3-A (생성 모드) |

---

## Step 2: 상품 데이터 로드

다음 파일들을 읽어서 상품의 시각적 컨텍스트를 파악한다:

1. `data/products/{slug}/meta.ts` — 선박명, 항로, 기항지 목록
2. `data/products/{slug}/intro-data.ts` — 크루즈 시설 분위기
3. `data/products/{slug}/schedule-days-data.ts` — 주요 관광지, 랜드마크 이름 (처음 100줄이면 충분)
4. `references/destination-visuals.md` — 해당 상품의 시각 아이덴티티 (색감, 대표 풍경, 분위기, 추천 시간대, 랜드마크, 소품/스타일링)

`destination-visuals.md`가 핵심이다. 여기서 얻는 색감·분위기·시간대 정보가 프롬프트의 분위기를 결정한다. 상품 데이터 파일에서는 구체적인 지명과 랜드마크 이름을 보충한다.

---

## Step 3-A: 장면 설계 (생성 모드 — 자연어 입력)

사용자의 자연어 설명을 **5요소 프레임워크**로 분해한다. `references/prompt-guide.md`를 참조하여 각 요소를 구성한다.

| 요소 | 설명 | 사용자 미지정 시 |
|------|------|------------------|
| **Style/Medium** | 시각 포맷 — cinematic photograph, photorealistic 등 | 기본: `A cinematic photograph` |
| **Subject** | 인물/피사체 — 외모, 의상, 포즈 | 사용자에게 질문 (인물 여부, 나이대, 인원 등) |
| **Setting** | 배경/환경 — 상품 목적지 기반 | `destination-visuals.md`의 대표 풍경 + 상품 데이터 랜드마크 활용 |
| **Action** | 행동/동작 | 사용자 설명에서 추출, 미지정 시 장면에 자연스러운 행동 |
| **Composition** | 구도/카메라/렌즈 | 기본: `35mm lens, shallow depth of field` |

**Subject 처리 원칙**: 인물이 포함된 장면이면 사용자가 인물 묘사를 제공할 때까지 기다린다. "커플", "가족" 같은 최소 힌트라도 있으면 진행하되, 구체적 외모(나이, 옷, 머리 등)는 사용자 지정을 우선한다. 인물 없이 풍경만 원하는 경우도 당연히 가능하다.

---

## Step 3-B: 장면 설계 (최적화 모드 — 기존 프롬프트)

기존 프롬프트를 5요소로 분석한 뒤, 상품 컨텍스트와 충돌하는 부분을 교체한다.

**교체 대상**:
- Setting → 상품 목적지의 풍경/배경으로 교체
- 색감/분위기 → `destination-visuals.md`에 맞게 조정
- 시간대/조명 → 사용자가 지정한 게 있으면 유지, 없으면 `destination-visuals.md` 추천 시간대 적용
- 소품 → 상품 분위기에 어울리는 소품으로 교체

**보존 대상**:
- Subject (인물 묘사) — 사용자가 명시적으로 변경 요청하지 않는 한 유지
- Style/Medium — 기존 스타일 유지
- Composition (카메라 설정) — 유지하되 나노바나나 최적화 적용

**나노바나나 최적화**: `prompt-guide.md`의 규칙으로 전체 프롬프트를 점검하고 보강한다.

---

## Step 4: 프롬프트 생성

`references/prompt-guide.md`를 읽고 다음 규칙을 적용한다:

### 필수 적용 사항
1. **서술형 문장만 사용** — 지시형("don't show X")이나 부정형 금지. 원하는 것만 묘사
2. **카메라/렌즈 명시** — 35mm, 50mm, macro 등 구체적 렌즈 타입
3. **조명 조건 구체화** — "bright midday sunlight", "golden hour warm backlight" 등
4. **품질 마커 추가** — photorealistic, 4K, highly detailed 중 2~3개
5. **대기/분위기 언어** — atmospheric language로 색감과 공기감 표현
6. **질감 요소** — natural film grain, subtle bokeh 등 시네마틱 질감

### 색감 반영
`destination-visuals.md`에서 해당 상품의 **색감 키워드**를 가져와 프롬프트 마지막에 color palette로 반영한다.
예: `warm saturated tropical color palette` (오키나와), `cool blue and white arctic color palette` (알래스카)

### 언어
영문으로 작성한다. 나노바나나는 영문 프롬프트에서 가장 정확한 결과를 생성한다.

### 프롬프트 구성 순서
```
[Style/Medium] + [Subject 상세 묘사] + [Action] + [Setting 상세 묘사] + [조명/시간대] + [소품/디테일] + [Composition/카메라] + [품질 마커] + [color palette]
```

---

## Step 5: 출력

### 생성 모드 (자연어 → 프롬프트)
마크다운 코드블록으로 프롬프트 1개를 출력한다.

```
[생성된 프롬프트]
```

### 최적화 모드 (기존 프롬프트 → 프롬프트)
프롬프트 1개 + 변경 포인트 요약 테이블을 출력한다.

```
[최적화된 프롬프트]
```

## 주의사항

- 상품 데이터에 없는 관광지/랜드마크를 임의로 추가하지 않는다. `schedule-days-data.ts`에 나오는 실제 지명만 사용한다.
- 인물 묘사는 사용자가 제공한 내용을 충실히 반영하며, AI가 임의로 인종/외모를 변경하지 않는다.
- 프롬프트 길이는 50~120단어를 목표로 한다. 너무 짧으면 디테일 부족, 너무 길면 나노바나나가 일부를 무시할 수 있다.
- 텍스트(글자)가 이미지에 포함되어야 하면 따옴표로 감싸고 25자 이내로 제한한다.
